   ```python
       """Record current portfolio value to database."""
       def _sync():
           cursor = db.cursor()
           # Compute total portfolio value
           total_value = compute_portfolio_value(cursor, price_cache)
           # Insert snapshot
           cursor.execute(
               """INSERT INTO portfolio_snapshots (id, user_id, total_value, recorded_at)
                  VALUES (?, 'default', ?, datetime('now'))""",
               (str(uuid.uuid4()), str(total_value)),
           )
           db.commit()
           return total_value
       
       return await run_in_threadpool(_sync)
   ```

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Portfolio calculation → Snapshot storage | Portfolio value computed from live prices (cache) and DB state; must remain consistent |
| Background task → Database connection | Long-running task accesses DB repeatedly; must not deadlock or corrupt state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Denial of Service | Snapshot task holds DB lock | **mitigate** | Each snapshot is a quick atomic transaction (write lock acquired, data inserted, released); 30-second interval is slow enough to not create contention with trade execution (single-user demo) |
| T-02-02-02 | Denial of Service | Background task not cancelled on shutdown | **mitigate** | Lifespan explicitly calls `snapshot_task.cancel()` and awaits the task with CancelledError handling; prevents orphaned tasks and "database is closed" errors |
| T-02-02-03 | Information Disclosure | Snapshot values reflect stale prices | **mitigate** | compute_portfolio_value() uses live prices from PriceCache (updated every 500ms by market data source); snapshots taken in quick transactions, not batched |
| T-02-02-04 | Tampering | Snapshot count inflated by concurrent recording | **mitigate** | Background task runs single-threaded (asyncio); only one snapshot recorded per iteration; snapshot immediately post-trade is in same transaction as trade (no duplicates) |

**Summary:** All ASVS L1 mitigations in place for background task lifecycle and data consistency.

</threat_model>
