4. **Create `frontend/store/priceStore.ts`** (Zustand store scaffold per D-03):
```typescript
import { create } from 'zustand'

export interface PriceUpdate {
  ticker: string
  price: number
  previous_price: number
  timestamp: string
  direction: 'up' | 'down' | 'flat'
  change: number
  change_percent: number
}

interface PriceState {
  prices: Record<string, PriceUpdate>
  history: Record<string, number[]>
  status: 'connecting' | 'live' | 'reconnecting'
  setPrice: (ticker: string, update: PriceUpdate) => void
  setStatus: (status: PriceState['status']) => void
}

export const usePriceStore = create<PriceState>((set) => ({
  prices: {},
  history: {},
  status: 'connecting',
  setPrice: (ticker, update) =>
    set((state) => ({
      prices: { ...state.prices, [ticker]: update },
      history: {
        ...state.history,
        [ticker]: [...(state.history[ticker] || []), update.price].slice(-60),
      },
    })),
  setStatus: (status) => set({ status }),
}))
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client ↔ Browser | Static export served by FastAPI; no dynamic server-side rendering in Phase 4 |
| Browser ↔ SSE/API | EventSource connects to `/api/stream/prices` (same origin); fetch() calls to `/api/*` |
| Store ↔ Component | Zustand state is client-side only; no PII or secrets stored |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Tailwind theme config | Accept | Config is static, not user-controlled; no risk |
| T-04-02 | Information Disclosure | Zustand store in DevTools | Accept | Store contains only live prices and status (public data); no secrets |
| T-04-03 | Spoofing | next.config.js static export | Accept | Static files served by backend; cannot be modified by client |

All security enforcement is delegated to backend (Phase 1-3). Frontend assumes API responses are trusted.
</threat_model>
