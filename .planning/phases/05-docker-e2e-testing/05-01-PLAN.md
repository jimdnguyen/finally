- RUN pip install uv
- COPY backend/pyproject.toml backend/uv.lock ./
- RUN uv sync --frozen --no-install-project (cacheable layer)

**Stage 3: Runtime (Python 3.12 slim)**
- FROM python:3.12-slim
- RUN useradd -m -u 1000 appuser

- COPY backend/app ./app
- COPY --from=frontend /build/out ./static
- ENV PATH="/app/.venv/bin:$PATH" PYTHONUNBUFFERED=1

- USER appuser
- EXPOSE 8000

**Current structure (add to if not present):**
```python
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from fastapi.responses import JSONResponse
import logging

# Import all routers
from app.portfolio import create_portfolio_router
from app.watchlist import create_watchlist_router
from app.chat import create_chat_router
from app.market import create_stream_router
from app.health import create_health_router

logger = logging.getLogger(__name__)

def create_app():
    app = FastAPI(
        title="FinAlly",
        description="AI Trading Workstation"
    )
    
    # Register all API routers BEFORE static files (they take priority)
    app.include_router(create_portfolio_router(), prefix="/api")
    app.include_router(create_watchlist_router(), prefix="/api")
    app.include_router(create_chat_router(), prefix="/api")
    app.include_router(create_stream_router(), prefix="/api")
    app.include_router(create_health_router(), prefix="/api")
    
    # Mount static files for Next.js SPA
    # html=True: serve index.html for any 404 on missing files
    # Enables Next.js client router to handle /dashboard, /portfolio, etc.
    app.mount("/", StaticFiles(directory="static", html=True), name="static")
    
    return app

app = create_app()
```

- Exact string: `StaticFiles(directory="static", html=True)` (SPA fallback enabled)
- Exact string: `app.mount("/", StaticFiles...` (mount at root)

If routers are already included in a lifespan context manager or startup task, ensure the StaticFiles mount is still the LAST statement in the app setup.

| Container image registry | Docker image is built locally; no external registry risk |
| Dockerfile input validation | All COPY commands reference project-local files (frontend/, backend/, db/) — no external downloads except base images |
| Static file serving | Next.js static exports are compiled locally and serve without code execution risk |
| API routes | All API routes already validated in Phases 1-3 (SQLite, portfolio, LLM); no new untrusted input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing | Docker base image (node:20-alpine, python:3.12-slim) | mitigate | Use official Docker Hub images; pin versions in Dockerfile; verify checksums with `docker pull --disable-content-trust=false` before production |
| T-05-02 | Tampering | Frontend static assets during COPY | mitigate | Hash frontend/out/ after Next.js build; compare in Docker layer; fail if hash mismatch |
| T-05-03 | Repudiation | No audit log for container startup | accept | Container startup logs via Docker (`docker logs`); sufficient for demo purposes |
| T-05-04 | Information Disclosure | Environment variables in container | mitigate | Use --env-file at runtime; do NOT hardcode OPENROUTER_API_KEY in Dockerfile; all secrets loaded from .env |
| T-05-05 | Denial of Service | Port 8000 exposed without rate limiting | accept | Single-user demo; no rate limiting required; Docker network isolation sufficient |
| T-05-06 | Elevation of Privilege | Root user in container | mitigate | Run as non-root user `appuser` (UID 1000); no sudo access; filesystem owned by appuser |

</threat_model>
