Create `backend/app/db/__init__.py` with three functions:

1. **get_connection()** → sqlite3.Connection
   - Opens connection to database file at `os.environ.get("DB_PATH", "db/finally.db")`
   - Sets `check_same_thread=False` (safe for FastAPI single event loop)
   - Sets `row_factory = sqlite3.Row` (enable column access by name)
   - Executes PRAGMA journal_mode=WAL (not just =WAL; capture output to confirm it's set)
   - Executes PRAGMA synchronous=NORMAL
   - Executes PRAGMA foreign_keys=ON
   - Returns connection

2. **init_db()** → sqlite3.Connection
   - Calls get_connection()
   - Reads schema.sql from same directory (use pathlib.Path(__file__).parent / "schema.sql")
   - Executes schema.sql using cursor.executescript()
   - Calls seed_data(conn) to populate defaults
   - Commits and returns connection

3. **seed_data(conn: sqlite3.Connection)** → None
   - Checks if default user exists: `SELECT id FROM users_profile WHERE id='default'`
   - If not, inserts: `INSERT INTO users_profile (id, cash_balance, created_at) VALUES ("default", 10000.0, datetime('now'))`
   - Checks if watchlist is empty for default user: `SELECT COUNT(*) as cnt FROM watchlist WHERE user_id='default'`
   - If empty, inserts 10 default tickers: AAPL, GOOGL, MSFT, AMZN, TSLA, NVDA, META, JPM, V, NFLX (each with unique UUID id, using uuid.uuid4())
   - Commits

Export: `__all__ = ["get_connection", "init_db", "seed_data"]`

Key constraints from CLAUDE.md (project instructions):
- Use `os.environ.get()` for DB_PATH, defaulting to "db/finally.db"
- WAL mode is non-negotiable (STATE.md critical pitfall #5)
- Decimal precision: seed_data hardcodes 10000.0 as float (acceptable for seed); downstream code will use Decimal when reading
- SQLite thread safety: `check_same_thread=False` is safe here because FastAPI runs async code in a single thread, and we use `run_in_threadpool` for all database calls

Pattern: Copy structure from RESEARCH.md lines 790–847 but use pathlib for schema_path.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User Input → Database | External data (watchlist tickers, trade requests) crosses into persistence layer; must validate before storage |
| Environment Variables → Database Path | DB_PATH env var determines where SQLite file is created; must be safe default |
| Async Routes → Sync SQLite | FastAPI event loop must not block on database I/O; use `run_in_threadpool` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | SQL Injection via ticker in watchlist | mitigate | Use parameterized queries (? placeholders) in all SQL; never concatenate user input into queries. Phase 1 uses raw SQL with ? params; assert with grep `ticker\)` patterns. |
| T-01-02 | Information Disclosure | Database file readable by other users | mitigate | SQLite file permissions: Docker container user has exclusive read/write; verify file mode 0600. Not applicable in Phase 1 (local dev); document for production deployment. |
| T-01-03 | Denial of Service | Malicious ticker added to watchlist millions of times | accept | Single-user demo; user is trusted. No rate limiting needed for v1. |
| T-01-04 | Tampering | Float precision loss in monetary calculations | mitigate | Store all monetary values (cash_balance, avg_cost, price, total_value) as TEXT. Read as Decimal, calculate in Decimal, convert to float only at JSON boundary. Verify with test_decimal_precision in Phase 1 Plan 02. |
| T-01-05 | Information Disclosure | .env file containing API keys committed to git | mitigate | .env not committed; .env.example is template. Verify via git check-ignore. Enforced by .gitignore. |

</threat_model>
