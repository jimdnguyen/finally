Create `backend/app/portfolio/service.py` with business logic functions:

1. **compute_portfolio_value(cursor: sqlite3.Cursor, price_cache: PriceCache) → Decimal**
   - Fetches cash_balance from users_profile WHERE id='default'
   - Fetches all positions WHERE user_id='default' AND quantity > 0 (ticker, quantity, avg_cost)
   - For each position, gets current price from price_cache.get(ticker)
   - Sums position values: stock_value += Decimal(str(qty)) * Decimal(str(current_price))
   - Returns cash + stock_value as Decimal
   - All intermediate values converted from database types or PriceCache to Decimal (initialize from strings)
   - CRITICAL: Initialize Decimal from strings, never from float

2. **get_portfolio_data(cursor: sqlite3.Cursor, price_cache: PriceCache) → dict**
   - Fetches cash_balance from users_profile WHERE id='default'
   - Fetches all positions WHERE user_id='default' AND quantity > 0
   - For each position:
     - Gets current_price and change_percent from price_cache.get(ticker)
     - Calculates unrealized_pnl = (current_price - avg_cost) * quantity (all in Decimal)
     - Converts Decimal values to float for JSON: float(cash_balance), float(quantity), etc.
   - Computes total_value via compute_portfolio_value()
   - Returns dict with keys: cash_balance (float), positions (list of dicts), total_value (float)
   - All monetary conversions happen here (Decimal → float); downstream code receives floats

3. **validate_trade_setup(db: sqlite3.Connection, ticker: str, side: str, quantity: Decimal, price_cache: PriceCache) → tuple[bool, str]**
   - Validates ticker format (alphanumeric, 1-5 chars, uppercase)
   - Validates side ("buy" or "sell", case-insensitive)
   - Validates quantity > 0
   - Gets current price from price_cache for ticker; if not found, return (False, f"No price available for {ticker}")
   - For "buy": fetch cash_balance, check if quantity * price <= cash_balance; if not, return (False, "Insufficient cash")
   - For "sell": fetch current position for ticker, check if quantity <= current_qty; if not, return (False, "Insufficient shares")
   - Return (True, "") if all validations pass
   - IMPORTANT: This is validation only; no database writes. Actual execution happens in trade endpoint (Phase 2).

Imports: `import sqlite3`, `from decimal import Decimal`, `from app.market.cache import PriceCache`, `from app.portfolio.models import ...`

All Decimal operations follow RESEARCH.md Pattern 4 (lines 470–498): initialize from strings via `Decimal(str(value))`, calculate in Decimal, convert to float only at JSON boundary.

Pattern reference: RESEARCH.md Pattern 3 (lines 305–450) shows full trade execution; this plan's service.py extracts the validation and valuation parts that both GET portfolio and future trade endpoint will use.

**Function: create_portfolio_router() → APIRouter**

Returns an APIRouter with prefix="/api/portfolio", tags=["portfolio"], containing two endpoints:

1. **GET /api/portfolio (mapped to "")**
   - Dependencies: db: sqlite3.Connection = Depends(get_db), cache: PriceCache = Depends(get_price_cache)
   - Logic: 
     ```python
     async def get_portfolio():
         async def _get_state():
             cursor = db.cursor()
             data = get_portfolio_data(cursor, cache)
             return data
         
         state = await run_in_threadpool(_get_state)
         return PortfolioResponse(
             cash_balance=state['cash_balance'],
             positions=[PositionDetail(**p) for p in state['positions']],
             total_value=state['total_value'],
         )
     ```
   - Returns PortfolioResponse (Pydantic will serialize to JSON)
   - Wraps sync database access with run_in_threadpool from fastapi.concurrency
   - No database writes; read-only

2. **GET /api/portfolio/history (mapped to "/history")**
   - Dependencies: db: sqlite3.Connection = Depends(get_db)
   - Logic:
     ```python
     async def get_portfolio_history():
         async def _get_history():
             cursor = db.cursor()
             cursor.execute(
                 "SELECT total_value, recorded_at FROM portfolio_snapshots WHERE user_id='default' ORDER BY recorded_at ASC"
             )
             rows = cursor.fetchall()
             snapshots = [
                 {'total_value': float(row[0]), 'recorded_at': row[1]}
                 for row in rows
             ]
             return snapshots
         
         snapshots = await run_in_threadpool(_get_history)
         return PortfolioHistoryResponse(snapshots=snapshots)
     ```
   - Returns PortfolioHistoryResponse with list of snapshots
   - All total_value fields converted to float from database
   - Order by recorded_at ascending (oldest first, newest last) for chart display

Critical pattern from RESEARCH.md Pattern 2: Every database call is wrapped in `async def _sync_function(): ... await run_in_threadpool(_sync_function)` to avoid blocking event loop.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PriceCache → Portfolio Calculation | Portfolio value depends on live prices; must handle missing prices gracefully |
| Database Read → Response | Portfolio data must be consistent snapshot (no partial reads across multiple queries) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | Invalid ticker crashes portfolio endpoint | mitigate | Validate ticker format (1-5 chars, alphanumeric) in validate_trade_setup(); if price not found in cache, skip position. Phase 1 tests: test_get_portfolio with invalid ticker should handle gracefully. |
| T-02-02 | Information Disclosure | Portfolio value leaks via timing attack | accept | Single-user demo; no multi-user data isolation needed. |
| T-02-03 | Tampering | Float precision loss in P&L calculation | mitigate | Use Decimal for all arithmetic; convert to float only at JSON boundary. Test with test_decimal_precision verifying exact values. |
| T-02-04 | Denial of Service | Massive portfolio (1M positions) causes slow response | accept | Single-user demo with ~50 tickers; scaling not a concern for v1. |

</threat_model>
