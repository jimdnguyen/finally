     * Try: call add_watchlist_ticker() or remove_watchlist_ticker() (will be in watchlist/service.py)
     * On success: append to executed_actions["watchlist_changes"]
     * On Exception: append error message to executed_actions["errors"], continue
   - Return {"trades": [...], "watchlist_changes": [...], "errors": [...]}
   - Docstring: "Execute trades and watchlist changes from LLM response; continue on failure"

7. execute_chat(db: sqlite3.Connection, user_message: str, price_cache: PriceCache) -> dict:

   - Check os.environ.get("LLM_MOCK") == "true"; if yes, call execute_chat_mock()
   - Otherwise:
     a. Build fresh portfolio context (D-03): context = build_context_block(db.cursor(), price_cache)
     b. Load conversation history: history = load_conversation_history(db.cursor(), limit=10)
     c. Build system prompt: "You are FinAlly, an AI trading assistant..." + context (D-03)
     d. Call LLM: llm_response = call_llm_structured(system_prompt, history + [{"role": "user", "content": user_message}], ChatResponse)

   - On ValidationError from call_llm_structured: catch and return fallback response with error in actions
   - Return dict: {"llm_response": ChatResponse, "executed_actions": dict, "error": str or None}
   - Wrap entire execution in run_in_threadpool() since DB access is sync
   - Docstring: "Execute chat request: context injection, LLM call, auto-execution, persistence"

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User Input → Chat Message | Chat message from `/api/chat` request (ChatRequest validation) |
| LLM Response → Validation | JSON from OpenRouter (ChatResponse schema validation) |
| Validated Schema → Trade Execution | Trade fields from validated ChatResponse (re-uses execute_trade validation) |
| LLM Injection Attack | System prompt contains portfolio context; user message might attempt injection |
| Database Read/Write | Multiple DB accesses (context build, history load, message save) in single request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | LLM JSON → Trade Execution | Mitigate | Validated ChatResponse schema before execute_llm_actions() processes trades; re-uses execute_trade() validation (sufficient cash, sufficient shares) |
| T-03-06 | Information Disclosure | Portfolio Context in Memory | Accept | Context string built per request, not persisted; only LiteLLM call sees it (via HTTPS); single-user demo, no real PII |
