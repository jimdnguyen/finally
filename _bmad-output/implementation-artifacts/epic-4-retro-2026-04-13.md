The `|| true` pattern ensures Makefile targets are safe to run multiple times:

## What Challenged Us

### HSTS Preload List (Story 4.3)
Chrome hardcodes certain hostnames (`app`, `dev`, `test`) to force HTTPS. Our Docker service named `app` caused `ERR_SSL_PROTOCOL_ERROR` in Chromium.

**Root cause:** Chrome's HSTS preload list silently upgrades `http://app:8000` to `https://app:8000`

**Fix:** Renamed Docker service from `app` to `webapp`

**Detection time:** ~4 hours debugging; Firefox working fine was the clue

### UV Installation in Docker (Story 4.1)
`COPY --from=ghcr.io/astral-sh/uv:latest /uv /bin/uv` requires GitHub Container Registry authentication that Docker Desktop may not have configured.

**Fix:** Use `pip install uv` instead — works without auth, same result

### Container Path Resolution (Story 4.1)
`Path(__file__).parent` resolves differently inside containers because working directories change between dev and container runtime.

**Fix:** Added `DATABASE_PATH` environment variable set explicitly in Dockerfile

### Firefox Headless SSE Flakiness (Story 4.3)
EventSource connections in Firefox headless don't reliably report state. Tests verify StatusDot visibility instead of asserting green color.

**Trade-off:** Accepted — reliability over precision. Would need Chromium coverage for color assertions.

### CSS Hover in Headless (Story 4.3)
`group-hover:block` CSS patterns don't trigger on programmatic `.hover()` in headless browsers.

**Fix:** Used `dispatchEvent('click')` directly instead of hover → click sequence

## Key Learnings

1. **HSTS preload is invisible** — Browser silently upgrades HTTP to HTTPS for certain hostnames. Docker service naming matters.

2. **Container paths ≠ development paths** — Use environment variables for runtime paths, not `Path(__file__).parent` calculations.

3. **Browser automation needs system privileges** — `init: true`, `ipc: host`, `SYS_ADMIN` capability are required for stable Playwright in Docker.

4. **Test isolation = environment isolation** — CI matrix pattern (fresh containers per browser) prevents state pollution.

5. **Idempotency is a feature** — Makefile targets should always be safe to run twice with `|| true` guards.
