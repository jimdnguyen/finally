## Story-by-Story Debug Log

**2-1 — Portfolio API & Trade Execution**
- Python monkeypatch module reference bug: `monkeypatch.setattr('backend.db.DB_PATH', ...)` patches the wrong namespace. Must patch at the import site: `monkeypatch.setattr('backend.routes.portfolio.DB_PATH', ...)`.
- Rule: patch where the name is *used*, not where it is *defined*.

**2-3 — Positions Table & Portfolio Header**
- Zustand `EMPTY_POSITIONS` selector: inline `[]` in a selector creates a new reference on every render → infinite re-render loop. Fix: hoist to a module-level constant.

**2-4 — Trade Bar UI**
- `apiFetch` error shape mismatch: frontend expected `body.error`, backend returned `body.message`. Contract mismatch caught in testing.

**2-5 — Watchlist Management UI**
- `fetch` on a 204 No Content response throws if `.json()` is called on empty body. Added 204 guard before parsing.

**2-7 — P&L History Chart**
- Conditional rendering bug: LWC mounted before data arrived → received empty dataset it couldn't recover from. Fix: return skeleton if `!data || data.length === 0`.

**Post-epic visual bugs (found in live browser testing, same session)**
1. Main chart x-axis showed "1" repeatedly — sequential integers treated as Unix epoch (Jan 1 1970). Fixed by switching to real `Date.now()/1000` with dedup guard.
2. X-axis showed only hour ("13") — added `timeVisible: true, secondsVisible: true` to LWC timeScale config.
3. Main chart froze after 200 points — `SPARKLINE_CAP = 200` + `slice(-200)` keeps array length constant; `points.length > prevLengthRef.current` never fires. Fixed by tracking last timestamp instead of array length.
4. `WatchlistRow.tsx` — layout broken: flex layout caused price/change % to overflow or wrap. Replaced with CSS grid (`grid-cols-[2.5rem_1fr_auto]`) for stable column alignment.
5. `SparklineChart.tsx` / `PnLHistoryChart.tsx` — conditional rendering gaps: charts mounted before data was ready, causing LWC errors on empty datasets.
6. `useSSE.ts` + `backend/app/main.py` — SSE reconnection and backend streaming edge cases cleaned up.

## What Was Hard

- **jsdom gap**: Unit tests passed 100%, then 6 visual bugs appeared in the first live browser run. jsdom cannot simulate charting libraries, CSS animations, ResizeObserver, or LWC timestamp parsing.
- **LWC timestamp sharp edges**: Sequential integer timestamps = Jan 1 1970 epoch. 200-point sparkline cap freeze. Empty dataset behavior. Library-specific gotchas not obvious from documentation.
- **Python module patching**: Classic pytest/monkeypatch rule that will bite any developer who doesn't know it.
- **Accumulating technical debt**: Floating-point rounding, concurrency guards, decimal UX — each individually deferrable, collectively a list that needs tracking.

## Patterns Identified

1. **jsdom vs browser gap**: Visual components — especially those using charting libraries, CSS transitions, ResizeObserver, or requestAnimationFrame — require real browser verification. Unit tests are necessary but not sufficient.

2. **LWC time format assumptions**: Numbers passed as timestamps are Unix epoch seconds. Sequential ordinals fail silently (render as 1970 dates). Always use real wall-clock timestamps with a dedup guard when feeding LWC series data.

3. **Python patch-at-import-site rule**: When using `monkeypatch.setattr`, the path must be the module where the name is *imported and used*, not where it is originally defined.
