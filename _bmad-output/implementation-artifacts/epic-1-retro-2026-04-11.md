## What Could Be Improved

- **Context7 usage**: Library APIs (LightweightCharts v5, Tailwind v4) caused issues when relying on training data. Should proactively use Context7 to fetch current docs before coding against any library.
- **Visual/integration testing gap**: Two bugs (CSS specificity, SSE dev proxy) passed all unit tests but were visible immediately when running the app. Unit tests in jsdom don't render CSS or handle cross-origin networking.
- **DB_PATH configuration**: Scattered across 3 backend modules, requiring 3 separate monkeypatches in tests. Should be centralized.
- **Defensive coding flags in reviews**: Code reviews flagged missing error handling that violates project standards ("validate at system boundaries only"). Need to calibrate review criteria to match project conventions.

## Bugs Found During Retrospective

### Bug 1 — SSE Not Connecting in Dev Mode
- **Root cause**: `EventSource('/api/stream/prices')` in `useSSE.ts` hits the Next.js dev server (port 3000), not the backend (port 8000). No proxy/rewrites configured in `next.config.ts`.
- **Fix**: Added `rewrites` to `next.config.ts` to proxy `/api/:path*` to `http://localhost:8000/api/:path*`. Only active during `next dev`; production serves both from same origin.

### Bug 2 — Blue Left Border Not Visible
- **Root cause**: Tailwind v4 CSS specificity conflict. `border-border` and `border-blue-primary` both set `border-color` for ALL sides. Stylesheet order made `border-border` win, hiding the blue active indicator.
- **Fix**: Changed to side-specific utilities: `border-b-border` (bottom separator), `border-l-blue-primary` / `border-l-transparent` (left active indicator).
